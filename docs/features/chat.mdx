## 概述

NexusMind 的 AI 對話基於 **Vercel AI SDK v6** 實現串流對話，預設使用 Google **Gemini 3 Flash Preview** 模型，亦可切換至 **Gemini 3.1 Pro Preview**（進階推理）或 **Gemini 3.1 Flash Lite Preview**（輕量快速）。與一般 AI 聊天機器人的最大差別是：NexusMind 會自動引用你的知識庫來回答問題（RAG 增強），讓每個回答都有根據、可溯源。

| 項目 | 值 |
|------|-----|
| 預設模型 | `gemini-3-flash-preview`（可切換 Pro / Lite） |
| 最大推理步驟 | 5 步（含 Google Search 工具呼叫） |
| 路由超時 | `maxDuration = 300` 秒 |
| 串流延遲 | 5ms（Unicode grapheme 分割，支援中文逐字流） |

## 開始對話

1. 從左側導航欄點擊 **Chat** 進入對話頁面
2. 在輸入框輸入你的問題
3. AI 會自動檢索知識庫並結合檢索結果回答

## Adaptive RAG 三層檢索

NexusMind 使用三層檢索策略，確保找到最相關的資訊：

1. **向量搜尋**（pgvector）— 使用 `gemini-embedding-001`（768 維）進行語義相似度比對
2. **BM25 關鍵字搜尋** — 基於 `tsvector` 的全文搜尋，精準找到特定術語
3. **LightRAG 圖查詢** — 透過知識圖譜的實體關係檢索（hybrid 模式），發現隱含的關聯

三層結果使用**倒數排名融合（RRF）**合併，`RRF_K = 60`。

### RAG 閾值設定

| 參數 | 值 | 說明 |
|------|-----|------|
| `RAG_MATCH_THRESHOLD` | 0.6 | 全域向量匹配閾值 |
| `DOC_SECTION_MATCH_THRESHOLD` | 0.55 | 單文件模式閾值 |
| `RSS_MATCH_THRESHOLD` | 0.45 | RSS 來源（多篇拼接，語意稀釋）|
| `HIGH_SIMILARITY_THRESHOLD` | 0.78 | 超過此值跳過 LLM grading |
| `RAG_MATCH_COUNT` | 12 | 每次搜尋返回的結果數 |
| `MAX_RELEVANT_DOCUMENTS` | 3 | 最終返回的相關文件數 |

### 動態上下文預算

系統根據查詢複雜度自動調整檢索範圍：

| 查詢類型 | 條件 | maxDocs | maxChunksPerDoc |
|---------|------|---------|-----------------|
| 複雜分析 | 含分析關鍵字 | 5 | 10 |
| 中等查詢 | 超過 50 字元 | 3 | 6 |
| 簡短查詢 | 少於 10 字元 | 1 | 3 |
| 預設 | 其他 | 3 | 6 |

### 超時保護

| 層級 | 超時 |
|------|------|
| RAG 總超時 | 12,000 ms |
| LightRAG 圖查詢 | 5,000 ms |
| Grading + Retry | 5,000 ms |
| 記憶檢索 | 5,000 ms |

所有檢索均帶超時保護，任何一層失敗不影響主對話流程。

## 系統提示詞組成

AI 的系統提示詞由 9 個層級依序組合：

| 層級 | 名稱 | 說明 |
|------|------|------|
| 1 | 基礎提示 | 注入當前日期、人格定義、Grounding 策略 |
| 2 | 回應風格 | 根據使用者訊息推斷風格（結構化/簡潔/Markdown）|
| 3 | 知識提示 | 有 RAG 結果時注入引用格式標準 |
| 4 | 知識品質信號 | 每份文件的可信度與新鮮度評估（內部參考）|
| 5 | 自訂指令 | 使用者的自訂系統提示（`profiles.system_prompt`）|
| 6 | 記憶上下文 | 已儲存的事實、偏好、行為 |
| 7 | 技能索引 | 使用者已啟用的技能清單 |
| 8 | 新鮮度指令 | 必要時強制 Google Search |
| 9 | Markdown 格式 | 結構化輸出指引 |

### 知識品質評估

每份檢索到的文件會標註：

- **可信度**：高（語意高度吻合）、中（相關但非核心）、低（部分相關）
- **新鮮度**：最新（≤30 天）、近期（≤180 天）、偏舊（>180 天）
- **來源加權**：內部文件 +0.04、外部文件 -0.02

## AI 記憶系統

系統自動從對話中提取並記住使用者的偏好與行為模式。

### 記憶提取分類

| 分類 | 重要度 | 觸發模式範例 |
|------|--------|------------|
| 偏好 | 0.7 | 「我喜歡…」「我偏好…」「我習慣…」|
| 事實 | 0.6 | 「我是…」「我在…」「我的…」|
| 行為 | 0.5 | 「每次都…」「通常我會…」「我經常…」|

### 記憶去重

使用三層去重策略（Bigram Jaccard 相似度閾值 0.6）：
1. 子字串包含檢查
2. Bigram 相似度檢查（≥ 0.6）
3. 過濾重複內容

### 記憶檢索

| 參數 | 值 |
|------|-----|
| 搜尋閾值 | 0.5 |
| 最大結果數 | 10 |
| 加權公式 | `similarity × 0.7 + importance × 0.3` |

## RAG 透明度面板

每次 AI 回答後，你可以展開「引用來源」面板，查看：

- AI 參考了哪些文件
- 每個引用的相關性分數與可信度
- 檢索方法（local / web / hybrid）
- 是否使用了圖譜上下文
- 原始查詢與改寫次數

透明度資料儲存在 `messages.metadata.rag_transparency` 中。

## 對話功能

| 功能 | 說明 |
|------|------|
| **對話分叉** | 從某個訊息開始建立新的對話分支（`parent_conversation_id` + `fork_from_message_id`）|
| **對話匯出** | 匯出為 DOCX（docx v9）、PPTX（pptxgenjs）、PDF（Playwright 渲染）|
| **Inline Suggestions** | 對話後自動生成 1-3 個快速追問（15 字以內，8 秒超時）|
| **思考進度動畫** | 即時顯示 AI 的推理步驟與經過時間 |
| **提示詞庫** | 儲存和重用常用的提示詞模板（支援公開分享）|
| **助理預設** | 5 種內建角色：一般助手、寫作助手、程式助手、分析助手、創意助手 |
| **儲存為報告** | 將 AI 回覆儲存為 Canvas 報告（取首 60 字為標題）|

## 串流與持久化

### 串流配置

```
smoothStream:
  delayInMs: 5
  chunking: Intl.Segmenter（Unicode grapheme 分割）
```

### 訊息持久化策略

系統使用三層邏輯確保訊息正確儲存：
1. 最新訊息為 assistant 且已有內容 → 覆寫（isFinal）
2. 最新訊息為 assistant 但內容為空 → 補齊
3. 最新訊息為 user → 插入新的 assistant 訊息

## 完整流程

```
使用者訊息 → 並行啟動：
├─ RAG 查詢（12s timeout）
│  ├─ 向量搜尋 + BM25（RRF 融合）
│  ├─ LightRAG 圖查詢（5s timeout）
│  └─ Grading + Retry（5s timeout）
├─ 記憶檢索（5s timeout）
└─ 儲存使用者訊息（非阻塞）

組合系統提示詞（9 層）→ streamText → onFinish：
├─ 儲存最終回覆 + RAG metadata
├─ 非同步生成 Inline Suggestions
└─ 非同步記憶萃取（不阻塞回應）
```
