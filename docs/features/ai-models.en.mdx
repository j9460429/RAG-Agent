## Embedding Configuration

| Item | Value |
|------|-------|
| Model | `gemini-embedding-001` |
| Vector Dimensions | 768 |
| Storage | Supabase pgvector (HNSW index) |
| Index Parameters | m=16, ef_construction=64 |
| Similarity | cosine (`vector_cosine_ops`) |

Embeddings are generated in these scenarios:

1. **Document Upload** ‚Äî Each chunk generates a 768-dim vector
2. **RAG Query** ‚Äî User questions converted to vectors for document matching
3. **Memory Search** ‚Äî Semantic search of user memories

## Vision OCR Mechanism

When document parsing returns empty text (typically scanned PDFs or images), the system automatically activates Gemini Vision OCR:

```
Document upload
  ‚Üí Marker microservice parsing
  ‚Üí Result is empty?
    ‚îú‚îÄ No ‚Üí Use Marker result
    ‚îî‚îÄ Yes ‚Üí Gemini Vision OCR
          ‚Üí Result is empty?
            ‚îú‚îÄ No ‚Üí Use OCR result
            ‚îî‚îÄ Yes ‚Üí Use built-in Parser (final fallback)
```

**Formats supporting Vision OCR:**

| Format | Description |
|--------|-------------|
| `.png`, `.jpg`, `.jpeg` | Sent directly to Vision model |
| `.pdf` (scanned) | Fallback when Marker returns empty text |

## API Key Configuration

### Basic Setup

```bash
# .env.local
GOOGLE_GENERATIVE_AI_API_KEY=AIzaSy...
```

All Gemini models share a single API Key.

### OAuth Token Mode (Optional)

```bash
USE_OAUTH=true
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
```

One-time authorization required before use:

```bash
npx gemini init
```

OAuth mode is suitable for scenarios requiring higher quotas or Google Workspace integration.

## Streaming Configuration

AI chat uses Vercel AI SDK's `streamText` for streaming responses:

| Item | Value |
|------|-------|
| SDK | `ai` v6 (Vercel AI SDK) |
| Stream Delay | 5 ms |
| Chunking Method | `Intl.Segmenter` (Unicode grapheme) |
| Support | CJK character-by-character streaming, Emoji integrity |
| Route Timeout | 300 seconds (`maxDuration`) |
| Heartbeat Interval | 15 seconds (prevents connection timeout) |

### SSE Heartbeat Mechanism

Long-running research tasks periodically send heartbeat signals to prevent connection timeouts from reverse proxies (like Nginx) or NAS environments:

```
Every 15 seconds sends an SSE heartbeat (empty comment)
‚Üí Keeps HTTP connection alive
‚Üí Heartbeat timer cleared when task completes
```
